1. INTRODUCTION
This is a functional specification for the User Authentication Module.
The system provides single sign-on, multi-factor authentication, and role-based access control.

2. REQUIREMENTS
REQ-001: The system shall support SSO login via SAML 2.0 and OAuth 2.0.
REQ-002: The system shall enforce password complexity: minimum 8 characters, uppercase, lowercase, number, special character.
REQ-003: The system shall lock accounts after 5 consecutive failed login attempts.
REQ-004: The system shall support MFA via TOTP authenticator apps.
REQ-005: The system shall allow password reset via registered email address.

3. NON-FUNCTIONAL REQUIREMENTS
NFR-001: Authentication response time shall be under 200ms for 95th percentile.
NFR-002: All passwords shall be hashed using bcrypt with a cost factor of 12.
NFR-003: Session tokens shall expire after 30 minutes of inactivity.

4. ACCEPTANCE CRITERIA
All requirements must be verified through automated integration tests.
Load testing must demonstrate compliance with NFR-001 under 1000 concurrent users.
